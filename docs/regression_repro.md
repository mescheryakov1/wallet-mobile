# Минимальный сценарий воспроизведения регресса

Регресс связан с тем, что `pubspec.lock` был пересобран на устаревшей версии Flutter/Dart (Dart 3.5, Flutter 3.24), что привело к даунгрейду базовых зависимостей и несовпадению с минимальной версией SDK, указанной в `pubspec.yaml` (Dart >= 3.9).

## Android
1. Переключиться на коммит после проблемного мержа (`7184f7a`).
2. На среде с Flutter 3.35+ (Dart 3.9+) выполнить `flutter pub get --enforce-lockfile`.
3. Получить ошибку/предупреждение о несовместимости lock-файла или попытку отката зависимостей до Dart 3.5 (из-за секции `sdks` в lock-файле), что ломает сборку/тесты.

## Windows
1. Те же шаги, что для Android, но в Windows-окружении (PowerShell или Git Bash).
2. `flutter pub get --enforce-lockfile` также сигнализирует о несогласованности SDK/зависимостей, поскольку lock-файл зафиксирован под Dart 3.5.

## Ожидаемый результат после фикса
- Lock-файл соответствует минимальным требованиям проекта (Dart >= 3.9, Flutter >= 3.35).
- `flutter pub get` на Android и Windows проходит без попыток даунгрейда зависимостей и без предупреждений о несовместимости SDK.
